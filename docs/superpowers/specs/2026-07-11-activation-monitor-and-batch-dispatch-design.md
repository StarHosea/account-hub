# 激活监控与会员批量出库设计

## 目标

- 批量激活页的“运行监控”准确展示所有进行中的激活账号，不受全量账号列表前 200 条限制。
- 会员账号页支持对当前勾选账号批量标记出库和撤销出库。

## 范围

- 批量操作只作用于当前勾选账号，不扩展到当前筛选结果或跨页选择。
- 批量出库操作仅在会员账号页显示，免费账号页保持不变。
- 复用现有 `/api/accounts/mark-used` 接口和前端 `handleMarkUsed` 处理逻辑，不新增重复接口。
- 不改变账号出库页面的发号和二次核验流程。

## 设计

### 激活运行监控

激活页轮询账号时向现有账号列表接口传入 `activation=activating`，由后端先筛选进行中账号，再应用分页。页面继续使用返回账号渲染邮箱和激活进度，并保留现有轮询频率、空状态和运行数量提示。

这避免了当前“先取全量账号前 200 条，再在浏览器中过滤”的截断问题。接口已有该筛选参数，因此不需要增加后端业务逻辑。

### 会员批量出库

会员账号页存在勾选项时，在列表工具栏显示两个操作：

- “标记出库”：确认后对所有勾选账号调用 `markAccountsUsed(tokens, true)`。
- “撤销出库”：确认后对所有勾选账号调用 `markAccountsUsed(tokens, false)`。

按钮复用现有忙碌状态、成功提示、错误提示和列表刷新逻辑。操作成功后，当前筛选条件可能使已变更账号离开列表，这是预期行为。

### 错误处理

- 未勾选账号时不展示批量按钮；现有处理函数仍保留空选择保护。
- 用户取消确认时不发请求。
- 接口失败时保留当前选择并显示现有错误提示。

## 验证

- 后端账号列表测试证明 `activation=activating` 会在分页前筛选进行中账号。
- 前端静态测试或组件测试证明会员页选择工具栏包含两个批量操作，并分别传递 `used=true` 和 `used=false`。
- 执行相关后端测试和前端生产构建。
- 在桌面与移动视口检查按钮不溢出、不遮挡，并确认免费账号页不出现会员出库操作。
