# 诊断报告模板

```markdown
## 诊断摘要
- 邮箱 / 时间 / 失败原因
- 卡点：`failed_step` + 最后 URL
- 分类：<超时|风控|selector|验证码|代理|状态机|其他>
- 存证：<有/无>

## 证据
- visible_ui（按钮/提示）
- manifest 末 2～3 步
- logs_tail 关键行

## 根因判断
- **置信度**：<已证实|高|低|无法判断>
- **判断**：（每条对应上文证据字段/行）
- **诊断缺口**：（置信度非已证实时必填）

## 修复建议
1. **立刻可做**（仅置信度 ≥ 高）
2. **需验证**（register-cdp-debug）
3. **基础设施**

## 闭环：增强采集
- brief 是否足够：<是 / 否 — 缺什么>
- 代码/Skill 改动：（具体文件与字段）
- 验证：（重跑失败 / 看新 brief 字段）

## 下一步
- [ ] 改代码 / 改 Skill / CDP / 重跑
```
